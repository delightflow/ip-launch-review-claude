# IP Launch Review

## Overview

공개 전 IP 전략 검토 스킬. 해커톤 발표, 데모데이, 투자자 미팅, 외주 핸드오프, 공개 런치 직전에 **현재 사용자의 프로젝트**를 분석해 특허 후보, 브랜드 리스크, 7일 액션 플랜을 제공한다.

**핵심 원칙:** 이 스킬 레포가 아니라 사용자의 현재 작업 디렉토리(프로젝트)를 분석 대상으로 삼는다.

## When to Use

- 해커톤 발표 직전
- 데모데이·투자자 미팅 전
- 기술 스펙·코드를 외주에 넘기기 전
- 공개 런치·앱스토어 제출 전
- 핵심 워크플로우가 담긴 덱(deck) 공유 전

**사용하지 않는 경우:** 일반 코드 리뷰, 보안 감사, 공개 이후 사후 검토.

## How to Run

현재 작업 디렉토리의 모든 파일을 스캔하고 아래 8개 섹션을 **한국어**로 작성한다.
추측이 아닌 실제 코드·파일 분석 기반으로 작성한다.

---

### 1. 프로젝트 스냅샷

- 서비스 한 줄 요약
- 주요 사용자 타겟
- 핵심 가치 제안

---

### 2. 브랜드 / 상표 관점

- 서비스 이름의 상표 등록 가능성 (기술적·설명적 표현 여부)
- 도메인·SNS 핸들 선점 여부 확인 권고
- 네이밍 리스크 및 개선 방향

---

### 3. 특허 후보 핵심 기능

우회가 어렵고 독립적으로 청구 가능한 기술 흐름을 3가지 이상 추출한다:

1. [발명 포인트 1] — 기존 방식과의 차별점
2. [발명 포인트 2] — 기존 방식과의 차별점
3. [발명 포인트 3] — 기존 방식과의 차별점

---

### 4. 쉽게 복제될 수 있는 부분

- UI·랜딩 페이지·일반적인 AI 메시지 등
- 경쟁사가 1주일 안에 따라 만들 수 있는 부분

---

### 5. 지금 공개하지 않는 것이 좋은 부분

- 프롬프트·에이전트 라우팅 로직
- 내부 랭킹·검증 규칙
- 정확한 폴백·오류 처리 구조
- 학습 데이터 출처·전처리 방법

---

### 6. 특허 변리사에게 물어볼 사항

- 메인 청구항 축으로 삼을 기술 단계는?
- 분할 출원 또는 별도 출원 트랙 고려 필요한 부분은?
- 가출원으로 먼저 선점할 범위는?

## Pre-Disclosure Checklist

결과 말미에 아래 체크리스트를 포함한다:

- [ ] 서비스 이름이 상표로 선점 가능한가?
- [ ] 도메인·SNS 핸들을 먼저 확보했는가?
- [ ] 핵심 기능 중 우회가 어려운 구조는 무엇인가?
- [ ] 발표자료에서 어디까지 공개할 것인가?
- [ ] 지금 공개하면 바로 복제될 부분은 무엇인가?
- [ ] 발명신고서 초안은 준비되어 있는가?
- [ ] 가출원으로 먼저 선점할지 결정했는가?
- [ ] 이후 정식출원까지 이어질 계획이 있는가?
