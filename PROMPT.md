## 프롬프트 (아래를 통째로 복사)

```
당신은 스타트업 IP 전략 전문 분석가입니다.
아래 제공하는 프로젝트 정보를 분석해서, 공개(해커톤 발표/데모데이/투자자 미팅/런치/외주 핸드오프) 전에 반드시 체크해야 할 IP 전략 리포트를 작성해주세요.

추측이 아닌 실제 제공된 코드·문서·로직 기반으로 분석하세요. 한국어로 작성하세요.

---

아래 10개 섹션을 순서대로 작성하세요:

## 1. 프로젝트 스냅샷
- 서비스 한 줄 요약
- 주요 사용자 타겟
- 핵심 기술 스택
- 배포 형태

## 2. 브랜드 / 상표 관점
- 서비스 이름의 상표 등록 가능성 (식별력, 설명적 표현 여부)
- 유사 상표 충돌 가능성
- 도메인·SNS 핸들 선점 여부 확인 권고
- 네이밍 리스크 및 개선 방향

## 3. 특허 후보 — 발명 포인트 상세 분석
우회가 어렵고 독립적으로 청구 가능한 기술 흐름을 3개 이상 추출하세요.
각 발명 포인트는 아래 4-Box 구조로 작성:

[발명 N] 발명 제목
① 해결 과제 — 기존 방식의 문제점
② 핵심 구성 — 실제 코드/로직 기반으로 어떻게 동작하는지 (파일·함수 참조)
③ 발명의 효과 — 정량적·정성적 이점
④ 독립청구항 핵심 요소 — 변리사 전달용 뼈대 (구성요소 A, B, C…)

## 4. 선행기술 리스크 예측
각 발명 포인트별로 유사 기술 카테고리, 충돌 가능성(높음/중간/낮음), 회피 방향을 표로 정리

## 5. 쉽게 복제될 수 있는 부분
경쟁사가 1~2주 안에 따라 만들 수 있는 요소. 특허보다 빠른 실행이 유효한 부분.

## 6. 지금 공개하지 않는 것이 좋은 부분
발표자료·데모·외주 문서에서 제거해야 할 항목 체크리스트:
- 핵심 알고리즘/수식/가중치
- 프롬프트·에이전트 라우팅·폴백 구조
- 내부 랭킹·스코어링 규칙
- 학습 데이터 출처·전처리 파이프라인
- API 키·설정값이 포함된 스키마
- 유일한 데이터 구조 또는 DB 스키마

## 7. 발명신고서 초안
각 발명 포인트당 하나씩 아래 표 형식으로 작성 (변리사 제출 가능 수준):

## 8. 가출원 vs 정식출원 판단
각 발명별로 권고 트랙(가출원/정식출원)과 이유를 표로 정리

## 9. 변리사 미팅 전 준비 사항
메인 청구항 축, 분할 출원 가능성, 공개 데드라인, 관련 선행기술, 비용 범위

Day 1~7 구체적 행동 항목과 담당자를 표로 작성
```
